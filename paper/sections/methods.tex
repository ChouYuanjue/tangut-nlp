\section{Methods}

We design the comparison around the research questions rather than around every implementation variant. The repository contains the full ablation matrix; the main text keeps only the method families needed to answer whether strong prompting is already enough, what local signal helps most, and whether frontier/local complementarity can be turned into a workflow.

\subsection{Prompt-Only Upper Bounds}

The prompt-only family contains no weight updates. \texttt{baseline1} prompts the local model directly on the Tangut string, \texttt{baseline2} adds dictionary glosses, and \texttt{baseline2\_1\_cot} further decomposes generation into alignment, literal drafting, and rewriting. This family is closest to lexicon-augmented MT, except that the lexicon is injected through prompting rather than decoder constraints \cite{chen2017incorporating, hokamp2017lexically}.

We then add a deliberately strong frontier baseline based on DeepSeek-V3.2 with dictionary grounding and four fixed development-set exemplars spanning canonical, bodhisattva-practice, dharani-style, and transliteration-heavy titles. The prompt enforces compact title form, forbids explanatory paraphrase, and emits only the final title string. This baseline is intentionally strong: if frontier prompting already solves the task, the local-training story should shrink accordingly.

\subsection{Local Training Recipes}

We build 50k pseudo-Tangut/Chinese pairs from monolingual historical Chinese by mapping Chinese characters to Tangut characters through dictionary explanations. The four SFT branches differ only in how dictionary misses are handled and whether the target includes alignment structure:
\begin{itemize}
\item mixed-script keeps some Chinese characters in the source;
\item \texttt{UNK} replaces misses with \texttt{[UNK]};
\item semantic projection replaces misses through embedding lookup;
\item multitask keeps the \texttt{UNK} input but changes the target to \texttt{<dict\_match>\{...\}</dict\_match> <literal>\{...\}</literal> \{final title\}}.
\end{itemize}

Each synthetic set is combined with the 400 real training examples upsampled by a factor of 10. In the main narrative, \texttt{UNK}-SFT isolates the effect of input purity and multitask SFT tests whether explicit alignment structure helps further. The mixed and semantic variants remain in the repository but do not change the conclusions.

\subsection{Preference Alignment on Local Bases}

The DPO family asks a narrower question: once a strong local base exists, can reward-based alignment improve it without damaging title form? We apply DPO to an SFT base using reward-defined chosen/rejected pairs \cite{rafailov2023dpo}. For each training prompt we sample five outputs and score them with
\[
r(y) = \mathrm{lex}(y) - 0.01 \log \mathrm{ppl}(y),
\]
where $\mathrm{lex}(y)$ is the repository's lexical-coverage score. The reward therefore favors dictionary consistency with only a weak fluency tie-breaker, which is why we treat it cautiously later.

The legacy pipeline constructs preferences from the best pre-multitask SFT base, yielding the \texttt{final\_v2} checkpoint as a negative control. The follow-up DPO runs instead start from multitask SFT. After auditing the preference data, we keep only pairs whose reward gap exceeds either 0.2 or 0.4 and train both standard sigmoid DPO and a robust+weighted variant \cite{chen2024robustdpo,pang2024wpo}. This turns the DPO family into a controlled test of base-model choice, pair filtering, and loss robustness.

\subsection{Hybrid Adjudication}

The final method family operationalizes the paper's main positive claim. We build an inference-time adjudicator over the frontier DeepSeek baseline, UNK-SFT, and MT-DPO g0.4 sig. The closed adjudicator uses Azure-hosted GPT-5.4 and may either copy one candidate or make a minimal repair from candidate fragments. Its prompt treats the task as historical catalog-title reconstruction rather than free paraphrase, and the repository stores the exact templates and per-example JSON traces.

For reproducibility, we also add a deterministic open selector over the same 3-way pool. It uses title-suffix priors, candidate-length agreement, contamination penalties, conservative guards against truncation or expansion, and a small frontier prior. We further compare three simpler open controls over the identical pool: plurality vote, pairwise chrF-MBR, and a local Qwen2.5-7B reranker that can only copy one candidate verbatim. These baselines help separate task-aware conservative selection from generic agreement maximization.

\subsection{Supplementary EVOBC Portability Stress Test}

Reviewer questions naturally target portability across historical scripts rather than Tangut alone, so we add a second-script stress test built around EVOBC oracle-bone IDs. Each external EVOBC ID is compiled into a reversible private-use surrogate codepoint so that the Tangut dictionary and training pipeline can be reused without changing the single-character source contract. The transferable part of the method is therefore the symbolic interface and the local-training stack, not the Tangut codepoints themselves.

To move beyond a pure infrastructure port, we derive a real held-out probe from OBIMD sentence-level reading sequences by canonicalizing them into the EVOBC inventory through shared modern-character labels and sampling length-balanced 50/50/200 train/dev/test splits. This yields a deliberately conservative decipherment stress test rather than a second short-title benchmark. The oracle setup reuses the symbolic interface and synthetic local-training stack, but not the Tangut-specific reranker or reference-aware judge.

One evaluation detail matters. The oracle multitask SFT checkpoint still emits \texttt{<dict\_match>}, then \texttt{<literal>}, and then a freer Chinese explanation, mirroring the Tangut pipeline. The canonicalized oracle probe instead targets only the compact normalized decipherment string. We therefore report the oracle SFT model primarily through the \texttt{<literal>} field and keep the raw assistant tail as a control for output-format mismatch.

Because the main Tangut claim is workflow-level complementarity rather than local single-model dominance, we also evaluate a deliberately simple open 2-way oracle selector over the dictionary-grounded prompt and the MT-SFT \texttt{<literal>} output. If this minimal selector can already recover extra correct items, then the complementary local-repair behavior has transferred in the sense relevant to the paper.

\subsection{Implementation Details}

All learned models start from Qwen2.5-7B-Instruct with LoRA adapters. SFT uses three epochs, learning rate $2\times10^{-4}$, batch size 4, gradient accumulation 4, rank-64 LoRA, and maximum sequence length 512. DPO uses two epochs, learning rate $5\times10^{-5}$, batch size 1, gradient accumulation 8, $\beta=0.1$, rank-32 LoRA, cosine decay, and the same sequence length; the robust variants add label smoothing and WPO-style weighting. The reference-aware judge is a supplementary evaluator, not a training signal. It uses Azure-hosted GPT-5.4 to score reference agreement, source faithfulness, title-style fitness, and overall quality after seeing the Tangut source, gold reference, and candidate simultaneously.
