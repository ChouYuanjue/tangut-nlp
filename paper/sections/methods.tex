\section{Methods}

We design the comparison around the research questions rather than around every implementation variant. The full ablation matrix remains in the repository; the main text keeps only the families needed to answer whether strong prompting is already enough, what local training signal remains useful, and whether frontier/local complementarity can be turned into a workflow.

\subsection{Prompt-only upper bounds}

We start with the strongest version of the prompt-only story. The local prompt baselines contain no weight updates: \texttt{baseline1} prompts the local model directly on the Tangut string, \texttt{baseline2} adds dictionary glosses, and \texttt{baseline2\_1\_cot} further decomposes generation into alignment, literal drafting, and rewriting. This family is closest to lexicon-augmented MT, except that the lexicon is injected through prompting rather than decoder constraints \cite{chen2017incorporating, hokamp2017lexically}.

We then add a deliberately strong frontier prompt-only baseline based on DeepSeek-V3.2 with dictionary grounding and four fixed development-set exemplars spanning canonical, bodhisattva-practice, dharani-style, and transliteration-heavy titles. The prompt emphasizes compact title form, forbids explanatory paraphrase, and emits only the final title string. Decoding uses temperature 0.0 with a 512-token cap. The exact prompt script and exemplar IDs are stored in the repository. The purpose is to make the objection as strong as possible: if frontier prompting already solves the task, the local-training story should shrink accordingly.

\subsection{Local training recipes}

If prompting is not the whole story, the next question is what kind of local supervision helps most. We build 50k pseudo-Tangut/Chinese pairs from monolingual historical Chinese by mapping Chinese characters to Tangut characters through dictionary explanations, fixing the synthesis seed to 42, and keeping source lengths between 4 and 64 characters. The four SFT branches differ only in how dictionary misses are handled and whether the target includes alignment structure: mixed-script keeps 30--70\% Chinese characters, \texttt{UNK} replaces misses with \texttt{[UNK]}, semantic projection replaces misses through Qwen2.5-0.5B embeddings plus FAISS lookup, and multitask keeps the \texttt{UNK} input but changes the target to \texttt{<dict\_match>\{...\}</dict\_match> <literal>\{...\}</literal> \{final title\}}. This gives the model both an alignment channel and a final title target in one sequence. These examples are not philologically faithful Tangut reconstructions; they are structured training proxies. Each synthetic set is combined with the 400 real training examples upsampled by a factor of 10 and shuffled with seed 42.

In the narrative that follows, two local variants matter most. \texttt{UNK}-SFT isolates the effect of input purity, while multitask SFT tests whether explicit alignment structure adds value beyond purity alone. The mixed and semantic variants remain part of the repository-level ablation matrix, but they do not change the main story.

\subsection{Preference alignment on local bases}

The DPO family asks a narrower question: once a strong local base exists, can reward-based alignment improve it without damaging title form? We apply DPO to an SFT base using reward-defined chosen/rejected pairs \cite{rafailov2023dpo}. We sample five outputs per training prompt at temperature 0.8 and top-$p$ 0.95, then score them with
\[
r(y) = \mathrm{lex}(y) - 0.01 \log \mathrm{ppl}(y),
\]
where $\mathrm{lex}(y)$ is the repository's lexical-coverage score. The key point is not the exact tokenizer details, but that the reward favors dictionary consistency with only a weak fluency tie-breaker. We then keep the top-scoring output as chosen and the bottom-scoring output as rejected whenever the reward gap is at least 0.05.

The legacy pipeline first selects the best SFT base among the mixed, \texttt{UNK}, and semantic variants using chrF++, then constructs preference pairs from that base. In the current snapshot that selector chooses the \texttt{UNK} model, yielding the legacy checkpoint \texttt{final\_v2}, which serves as a negative control. The follow-up DPO runs instead start from the stronger multitask SFT model. After auditing the preference data, we keep only pairs whose reward gap exceeds either 0.2 (1,996 pairs) or 0.4 (498 pairs), and we train both standard sigmoid DPO and a robust+weighted variant \cite{chen2024robustdpo,pang2024wpo}. This turns the DPO family into a controlled test of base-model choice, pair filtering, and loss robustness.

\subsection{Hybrid adjudication}

The final method family operationalizes the paper's main positive claim. The oracle analysis suggested that frontier and local candidates are complementary, but an oracle is not a deployable system. We therefore build a real inference-time adjudicator over the frontier DeepSeek baseline, UNK-SFT, and MT-DPO g0.4 sig. For each item it sees the Tangut source, dictionary glosses, the three candidate titles, their lengths, and simple prefix/suffix consensus hints.

The adjudicator uses Azure-hosted GPT-5.4 at temperature 0.0. Its prompt treats the task as historical catalog-title reconstruction rather than free paraphrase: dictionary glosses are clues rather than canonical modern titles, and short inputs are penalized for explanatory expansion. The model may either copy one candidate or perform a minimal repair using only candidate fragments or directly attested gloss segments. The repository stores the exact templates, frontier prompting script, fixed exemplar IDs, and per-example adjudication JSON, so the closed-model part of the workflow is at least auditable.

For reproducibility, we also add a deterministic open selector over the same 3-way pool. It never synthesizes text or calls an external model. It uses title-suffix priors, candidate-length agreement, contamination penalties, conservative guards against truncation or expansion, and a small frontier prior. A guarded switch replaces the frontier output only when a local candidate has materially stronger conservative evidence.

\subsection{Implementation details}

All learned models start from Qwen2.5-7B-Instruct with LoRA adapters so that differences are attributable to supervision strategy rather than backbone changes. SFT uses three epochs, learning rate $2\times10^{-4}$, batch size 4, gradient accumulation 4, rank-64 LoRA, and maximum sequence length 512. DPO uses two epochs, learning rate $5\times10^{-5}$, batch size 1, gradient accumulation 8, $\beta=0.1$, rank-32 LoRA, cosine decay, and the same sequence length; the robust variants add label smoothing 0.1 and WPO-style weighting. The full pipeline fixes seed 42 for synthesis, data combination, and DPO training.

The reference-aware judge is a supplementary evaluator, not a training signal. It uses Azure-hosted GPT-5.4 to score reference agreement, source faithfulness, title-style fitness, and overall quality on a 1--5 scale after seeing the Tangut source, gold reference, and candidate simultaneously. For cross-judge robustness, the repository also includes a local Qwen2.5-7B-Instruct judge with the same rubric. We use the Azure judge as the primary table because it is the stricter and less saturated scorer.
