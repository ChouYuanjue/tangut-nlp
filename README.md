# Tangut NLP (西夏文自然语言处理)

这是一个针对极低资源西夏文（Tangut）机器翻译的研究项目。本项目旨在探索在缺乏大规模高质量平行语料的情况下，如何利用有限的字典资源和预训练大型语言模型（LLMs），实现西夏文到现代中文的翻译。

## 🌟 项目背景

西夏文是一种已经消亡的文字，相关的数字化资源极其匮乏，传统的基于大规模平行语料的神经机器翻译（NMT）方法难以适用。本项目提出了一套完整的“低资源翻译基准与优化方案”，从零样本评估到基于字典增强的 RAG 模型，再到伪合成数据监督微调（SFT）以及基于直接偏好优化（DPO）的最终方案，全面考察了大模型在处理极低资源语言时的能力极限与优化路径。

## 📊 评估标准 (Evaluation Metrics)

在无法依赖传统 BLEU 或人类标注 Ground Truth 的情况下，本项目构建了以下四维评估矩阵：

1. **忠实度 (Lexical Coverage / Dictionary Yield) - 核心硬指标**
   计算生成的中文翻译中，包含了多少源西夏文字符对应的字典释义。基于字典建立反向映射集合与模型输出计算交集比例。
2. **流畅度 (Perplexity, PPL) - 质量约束**
   计算中文输出结果的语言模型困惑度（基于 Qwen2.5-0.5B），PPL 越低说明生成的翻译越符合现代或文言文的真实中文语法。
3. **形态相似度 (chrF++) - 软对齐指标**
   通过字符级 n-gram 匹配得分评估翻译。相较于按词匹配的 BLEU，chrF++ 对古文和高密度表意文字的宽容度更高。
4. **LLM-as-a-Judge (盲测打分)**
   使用 SOTA 大语言模型作为裁判，在“未遗漏语义”和“语序通顺”两个维度给出 1-5 分的定量评分。

## 🚀 实验方法与基准 (Baselines & Methods)

本项目包含三个 Baseline 方案和一个最终优化方案：

### Baseline 1: 纯零样本指令生成 (Zero-shot LLM)
- **目的**：测试预训练基座模型对西夏文的原生理解能力。
- **配置**：基于大尺寸指令微调模型（如 Qwen2.5-7B-Instruct），直接输入西夏文要求翻译。

### Baseline 2: 纯字典增强生成 (Dictionary-Augmented RAG)
- **目的**：测试纯粹的上下文符号注入（无权重更新）在解决语序分歧（西夏文SOV -> 中文SVO）时的极限。
- **配置**：解析西夏文字典，提取输入句子中每个字符的释义放入 Prompt 中辅助模型重排语序并翻译。

### Baseline 3: 伪合成语料监督微调 (Synthetic SFT)
- **目的**：测试即使仅有伪平行语料，模型能否学会语法结构和注意力对齐。
- **配置**：利用古代中文语料和西夏文字典反向合成数万条伪西夏文语料，进行全参数或 LoRA 监督微调。

### 最终方案: 字典约束的直接偏好优化 (SFT + DPO)
- **目的**：探索序列级全局奖励寻优能否彻底解决句法约束和幻觉问题。
- **配置**：利用 Baseline 3 训练出的 SFT 模型进行多样性采样，结合“忠实度”和“流畅度”构建打分函数，生成偏好数据对（Chosen/Rejected），最后进行 DPO 训练对齐。

## 📂 项目结构

```text
tangut-nlp/
├── configs/            # DeepSpeed等训练配置文件
├── data/               # 数据目录
│   ├── dictionary/     # 西夏文字典与奖励映射
│   ├── dpo/            # DPO 偏好对数据
│   ├── eval/           # 评估测试集
│   ├── raw/            # 原始语料库与网页抓取数据
│   └── sft/            # SFT 训练用合成与组合数据
├── eval/               # 评测脚本 (ChrF, 覆盖率, 困惑度, LLM Judge)
├── experiments/        # Baseline 验证与推理脚本
├── logs/               # 训练与运行日志
├── models/             # 本地存放的模型权重 (如 Qwen2.5 模型)
├── results/            # 各方法生成的翻译结果和评测指标
├── scripts/            # 自动化流程脚本 (如下载、环境配置、主流程运行)
├── src/                # 核心处理源码 (数据准备、模板生成、字典解析等)
└── state/              # Pipeline 各阶段完成状态标识
```

## 🛠️ 安装与使用指南

### 1. 环境准备

建议使用包含两张或以上 GPU 的服务器进行训练实验（支持 A100/A800）。

```bash
# 赋予脚本执行权限
chmod +x scripts/*.sh

# 设置 Conda 环境并安装依赖
./scripts/setup_env.sh
# 或直接通过 pip 安装
pip install -r requirements.txt
```

### 2. 模型下载

拉取基础的开源模型权重至 `models/` 目录：
```bash
./scripts/download_models.sh
```

### 3. 一键运行 Pipeline

项目包含各个实验阶段的完整流水线，执行以下脚本即可顺序跑通数据合成、基准评估和最终训练优化：

```bash
./scripts/run_pipeline.sh
```

Pipeline 会生成状态文件保存在 `state/` 目录下，中途中断后可以支持断点续跑。每个实验结果将被妥善保存在 `results/` 目录下并附带评测分数的并排比较对比（如 `comparison.csv`）。
