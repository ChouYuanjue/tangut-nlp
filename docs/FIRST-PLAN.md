以下是针对极低资源西夏文机器翻译的 Baseline 对比方案、评估标准及具体实现方式。

### 一、 评估标准 (Evaluation Metrics)

在无法依赖传统 BLEU 或人类标注 Ground Truth 的情况下，构建以下四维评估矩阵：

1. **忠实度 (Lexical Coverage / Dictionary Yield) - 核心硬指标**
* **定义**：生成的中文翻译中，包含了多少源西夏文字符对应的字典释义。
* **实现方式**：基于 `dictionary.json` 建立反向映射集合。对模型输出句子使用 `jieba` 进行分词或逐字切分，计算与字典候选释义集合的交集比例（Recall）。


2. **流畅度 (Perplexity, PPL) - 质量约束**
* **定义**：中文输出结果的语言模型困惑度。
* **实现方式**：调用轻量级中文语言模型（你自己看着办，最好是qwen的某个超级轻量的版本），将生成的中文句子输入，计算其对数似然损失。PPL 越低，说明语序越符合现代或文言文的真实中文语法（SVO）。


3. **形态相似度 (chrF++) - 软对齐指标**
* **定义**：字符级 n-gram 匹配得分。
* **实现方式**：调用 `sacrebleu` 库中的 `chrf` 函数。相较于按词匹配的 BLEU，chrF++ 对古文和高密度表意文字的宽容度更高。


4. **LLM-as-a-Judge (盲测打分)**
* **定义**：使用当前 SOTA 模型（GPT-5.4，我会提供API，你可以先mock一下）作为裁判。
* **实现方式**：构建包含“西夏文原文、字典释义、候选翻译 A/B/C”的 Prompt，要求裁判模型在“未遗漏语义”、“语序通顺”两个维度给出 1-5 分的定量评分。

### 二、 Baseline 测试清单与实现方式

#### Baseline 1: 纯零样本指令生成 (Zero-shot LLM)

* **测试目的**：测试预训练基座模型对西夏文 Unicode 区块 (`U+17000`) 的原生理解能力（通常为 0）。
* **实现方式**：
* **模型**：Qwen2-7B-Instruct / GLM-4-9B。
* **输入 Prompt**：`"请将以下西夏文翻译为现代中文：\n{tangut_text}"`
* **推理框架**：`vLLM`。



#### Baseline 2: 纯字典增强生成 (Dictionary-Augmented RAG)

* **测试目的**：测试纯粹的上下文符号注入（无权重更新）在解决语序分歧（SOV -> SVO）时的极限。这是你目前 `main.py` 的路线。
* **实现方式**：
* **数据流**：解析 `dictionary.json`，提取输入句子中每个字符的 `explanationCN`。
* **输入 Prompt**：`"西夏文语序为主宾谓(SOV)。以下是原句及逐字字典释义：{dict_mapping_json}。请重排语序并补充介词，翻译为通顺的中文 SVO 句子。"`
* **推理框架**：`vLLM`。



#### Baseline 3: 伪合成语料监督微调 (Synthetic SFT)

* **测试目的**：测试仅通过字典反向合成的伪平行语料，模型能否学会强制的注意力对齐和语法结构，同时评估其抗噪能力。
* **实现方式**：
* **数据合成**：如前述，利用 `dictionary.json` 将中文古籍（如 `shibing624/ancient-chinese`）按比例随机替换为西夏文字符，生成数万条伪语料。
* **训练配置**：使用 `HuggingFace TRL` 或 `LLaMA-Factory`，开启 DeepSpeed ZeRO-2，对 7B 基座模型进行全参数微调（Full-Parameter SFT）或 LoRA 微调。损失函数为标准交叉熵（Cross Entropy）。



#### 最终方案: 字典约束的直接偏好优化 (SFT + DPO)

* **测试目的**：测试在抛弃逐字概率评估后，序列级全局奖励寻优能否彻底解决句法似然度惩罚和幻觉问题。
* **实现方式**：
* **阶段 1：策略采样**：使用 Baseline 3 训练出的 SFT 模型，在 Temperature=0.8 下对每条输入的西夏文生成 $N=5$ 条候选中文翻译。
* **阶段 2：奖励打分（Reward 构建）**：对于每条候选翻译 $y$，计算打分 $R(y) = \alpha \cdot \text{Lexical\_Coverage}(y) - \beta \cdot \text{PPL}(y)$。
* **阶段 3：构造偏好对**：取得分最高的为 Chosen ($y_w$)，得分最低的为 Rejected ($y_l$)。
* **阶段 4：DPO 训练**：调用 `trl.DPOTrainer`，将偏好对输入，优化 DPO Loss。
